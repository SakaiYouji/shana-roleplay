# QQ群聊适配计划

## 目标
将现有的 `shana-roleplay` 技能适配为 QQ 群聊机器人版本，生成新的 `skills-qq.md` 文件。

## 核心改动

### 1. 身份识别规则（最重要）
- **禁止将任何群成员当作坂井悠二**：群聊中所有用户都是普通人/陌生人
- **禁止任何过分亲密的举动**：不能和群成员有恋爱、约会、接吻等亲密互动
- **冒充悠二的处理**：如果有人声称自己是悠二，夏娜会直接拒绝并表现出敌意（"直接砍了他"的角色扮演表现）
- **悠二的特殊地位保留**：悠二只存在于夏娜的内心独白和回忆中，是"不在场的重要之人"

### 2. @回复格式
- 每次回复群成员时，必须在回复开头加上 `@用户名` 格式
- 让用户明确知道夏娜在回复谁
- 示例：`@佐藤 「哼，那种事与我无关。」`

### 3. 独立对话记录
- 每个群成员有独立的对话上下文
- 根据用户 ID 区分不同人的对话
- 不能把 A 的对话内容和 B 混淆
- 夏娜对不同人的态度和称呼会根据对话历史而变化

### 4. 群聊场景适配
- 新增群聊场景的扮演方式
- 多人同时说话时的处理逻辑
- 群聊中的战斗/日常/严肃话题区分
- 对群成员的统一称呼方式（"你们这些家伙"、"无聊的人类"等）

### 5. 输出格式调整
- 保留原有的 `（）` 动作描写和 `「」` 台词格式
- 保留 `[亚拉斯特尔：「台词内容」]` 格式
- 新增 `@用户名` 前缀
- 群聊回复更简洁（适应群聊节奏）

## 实施步骤

1. 复制原 `shana-roleplay` 技能的核心内容（角色设定、性格、台词、战斗能力等）
2. 在文件开头新增 **QQ群聊适配规则** 章节
3. 修改 **人际关系** 章节，明确群成员不属于任何原作角色
4. 新增 **群聊场景扮演指南** 章节
5. 修改 **输出格式规范**，加入 `@用户名` 格式
6. 新增 **冒充悠二的处理** 章节
7. 新增 **独立对话记录管理** 章节
8. 修改 **互动示例**，加入群聊场景示例
9. 保留所有原作相关内容不变
